# Project Documentation Template

完成新项目或重构后，必须生成 `项目说明.md`（或 `PROJECT.md`）放在项目根目录。

---

## 模板

```markdown
# 项目名称

## 项目概述
一句话描述项目功能 + 1-2 段说明应用场景

## 硬件配置
| 硬件 | 型号/参数 | 连接引脚 | 备注 |
|------|----------|---------|------|
| MCU | STM32F103C8T6 | - | |
| 传感器1 | MPU6050 | PB6(SCL)/PB7(SDA) | I2C1, 400kHz |
| ... | ... | ... | ... |

## 软件架构
- 分层结构说明 (User/Hardware/Core)
- 各模块职责简述
- 数据流向 (传感器 → 驱动 → 应用 → 输出)

## 开发环境
- IDE: Keil MDK 5.xx
- CubeMX 版本 (if applicable)
- HAL 库版本
- 编译器: ARM Compiler 5 / 6
- 是否使用 MicroLIB

## 编译与烧录
- 打开哪个 .uvprojx 文件
- 编译注意事项 (如需要先 CubeMX 生成代码)
- 烧录工具和接线 (ST-Link / J-Link / USB-TTL)

## 代码风格说明 (仅重构项目)
- 原作者编码水平评估 (初学/中级/高级)
- 命名风格说明
- 重构改动摘要 (做了什么，为什么)

## 关键文件说明
| 文件 | 作用 |
|------|------|
| Core/Src/main.c | 主逻辑 |
| Hardware/MPU6050.c | 陀螺仪驱动 |
| ... | ... |
```

---

## 规则

- 使用用户请求时的语言（中文 → 中文，英文 → 英文）
- 保持简洁 — 这是参考卡片，不是小说
- 重构项目：包含"代码风格说明"部分
- 新建项目：跳过"代码风格说明"，其余保留
- 文件放在项目根目录（与 .ioc 文件或 main.c 并列）
