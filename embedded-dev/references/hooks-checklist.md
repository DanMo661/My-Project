# Hooks 检查清单

## Hook 1：模块代码检查

**检查项目：**
- [ ] 语法检查：代码能否编译通过
- [ ] 任务完成度：是否实现了分配的功能
- [ ] 注释检查：是否有中文注释，注释是否清晰
- [ ] 代码风格：是否符合命名规范（App_/Com_/Int_ 前缀）
- [ ] 内存安全：是否有明显的内存泄漏或越界风险

**检查不通过处理：**
- 让子 agent 继续修改
- 最多重试 3 次

## Hook 2：编译检查

**检查项目：**
- [ ] 编译命令是否正确
- [ ] 编译输出日志是否正常
- [ ] 是否有 Error
- [ ] 是否有 Warning

**编译成功标准：**
- 0 Error(s), 0 Warning(s)

## Hook 3：整体检查

**检查项目：**
- [ ] 模块配合：各模块接口是否匹配
- [ ] 数据流：数据在各模块间是否正确传递
- [ ] 主逻辑：整体代码是否能完成用户需求
- [ ] 硬件初始化：引脚配置是否正确，外设初始化是否完整
- [ ] FreeRTOS 任务：任务优先级、栈大小、调度是否合理（若使用 FreeRTOS）

**检查不通过处理：**
- 分析原因，分配修复 agents
- 修改后重新编译（触发 Hook 2）

## Hook 4：编译失败修复

**处理流程：**
1. 分析编译错误类型和位置
2. 分配给合适的修复 agents
3. 修复后重新编译
4. 循环直到 0 Error 0 Warning

**最大重试次数：5 次**

**超过 5 次处理：**
- 向用户报告具体错误
- 请求用户提供更多信息
- 暂停自动修复流程
